# Tempo - Task Management API

Tempo — це мінімалістичний бекенд-сервіс для керування персональними або командними завданнями. Він надає простий та ефективний REST API для створення, читання, оновлення та видалення завдань, а також їх категоризації.

[![CI Pipeline](https://github.com/noirka/Task-Manager-Tempo-/actions/workflows/build.yml/badge.svg)](https://github.com/noirka/Task-Manager-Tempo-/actions/workflows/build.yml)

## Основна ідея

Мета проєкту — створити надійний і добре структурований API, використовуючи сучасні підходи до розробки Node.js, зокрема якісне налаштування інструментів та повне покриття тестами.

## Ключовий функціонал (Основні Сценарії)

- Управління Завданнями (Tasks):

  - Створення нового завдання з назвою, описом та терміном виконання.

  - Перегляд списку всіх завдань.

  - Перегляд деталей конкретного завдання.

  - Оновлення (редагування) існуючого завдання.

  - Позначення завдання як виконаного.

  - Видалення завдання.

- Управління Категоріями (Categories):

  - Створення, перегляд, оновлення та видалення категорій (наприклад, "Робота", "Особисте", "Терміново").

  - Призначення завдання до однієї або кількох категорій.

- Фільтрація та Сортування:

  - Фільтрація завдань за статусом (виконано/не виконано).

  - Фільтрація завдань за категорією.

  - Сортування за терміном виконання або датою створення.

## Технологічний Стек

- Платформа: Node.js

- Фреймворк: Express.js

- Мова: TypeScript

- База Даних: MongoDB або PostgreSQL

- Тестування: Jest

## Інструкції для Перевіряючого (Локальний Запуск)

Для перевірки функціоналу та проходження тестів (валідація CI) необхідні Node.js (v24+) та локальний або віддалений сервер MongoDB.

### Налаштування Середовища та Залежностей

1.  **Перейдіть у директорію проекту:**
    ```bash
    cd tempo-api
    ```
2.  **Встановіть залежності:**
    ```bash
    npm ci
    ```

### Запуск Тестів (CI Check)

Запустіть цю команду для виконання всіх юніт- та інтеграційних тестів:

```bash
npm test
```

Також для тестування було використано Artillery

Для оптимізації продуктивності були додані індекси MongoDB на поля `_id`, `isCompleted` та `createdAt`. Це значно покращило швидкість обробки запитів, пов'язаних з фільтрацією та пошуком.

| Метрика                    | Baseline (До оптимізації) | Optimized (Після індексів) | Покращення           |
| :------------------------- | :------------------------ | :------------------------- | :------------------- |
| **Успішних запитів**       | 1530                      | 2135                       | +39.5%               |
| **Median (час відповіді)** | 206.5 мс                  | 228.2 мс                   | _~0%_                |
| **P95 (час відповіді)**    | **9607.1 мс (9.6 сек)**   | **415.8 мс (0.4 сек)**     | **>95% прискорення** |

| **Головний висновок:** Час відповіді для 95% найповільніших запитів скоротився з 9.6 секунд до 0.4 секунди, підтверджуючи ефективність індексування бази даних.
